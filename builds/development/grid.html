Grid

The System

The "grid" system is essential to any framework. While the flexy system is not very different from how many others work, its philosophy does, if only ever so slightly.

The Shell

All page content is placed into a Flexy Shell. As the name implies, it serves as an empty "shell" within which you will place all of your modules. You should only need one shell per page and for semantics sake, shells should be applied to your pages main tag.

The Module

Modules house groups of similar content. A module serves as a sort of internal structure to a pages shell. Place your module divs within unique section tags.

The Unit

Units are where you place the actual content of your page. All headings, paragraphs, anchor tags, and etc.

On smaller viewports, units will take up 100% of the width, while on larger screens the unit will behave as follows:

Class: Unit

By default, units will take up even amounts of space across the width of the viewport.

Class: Unit

All units placed within a module will grow evenly as the page width increases.

Class: Unit

And will shrink evenly as the page width decreases.

Class: Unit-6

If you'd like to specify the width of a unit, simply add 1-10 to the unit in question. (Units within a single module should total to less than 10.)

Class: Unit

Adjacent units may also be left unspecified to simply take up the remaining space.
